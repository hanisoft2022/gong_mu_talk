## üìù Usage Examples

### ‚úÖ Correct Usage

```dart
// ‚úÖ Use semantic colors
Icon(Icons.check_circle, color: AppColors.success)

// ‚úÖ Use financial colors in calculator
Text(
  '‚Ç©1,234,560',
  style: TextStyle(color: AppColors.positive), // Green for income
)

// ‚úÖ Use emotional colors for actions
Icon(
  Icons.favorite,
  color: AppColors.like, // Pink for like button
)

// ‚úÖ Use theme colors when possible
Icon(
  Icons.info,
  color: Theme.of(context).colorScheme.primary,
)
```

### ‚ùå Incorrect Usage

```dart
// ‚ùå Don't use hardcoded Material colors
Icon(Icons.check_circle, color: Colors.green)

// ‚ùå Don't use Color() constructor directly
Container(color: Color(0xFFEF4444))

// ‚ùå Don't use arbitrary alpha values
Divider(color: Colors.grey.withOpacity(0.3))
```

## ‚ôø Accessibility

### Contrast Ratios

All color combinations meet WCAG 2.1 AA standards (minimum 4.5:1 for normal text, 3:1 for large text).

#### Text on Surface

| Combination | Ratio | WCAG AA | WCAG AAA |
|-------------|-------|---------|----------|
| Primary on White | 8.6:1 | ‚úÖ Pass | ‚úÖ Pass |
| Error on White | 4.5:1 | ‚úÖ Pass | ‚ùå Fail |
| Success on White | 3.8:1 | ‚ùå Fail* | ‚ùå Fail |
| Text Primary on Surface | 14.2:1 | ‚úÖ Pass | ‚úÖ Pass |

*Note: Success color should only be used for large text or icons, not body text.

#### Action Colors

| Combination | Ratio | WCAG AA | Use Case |
|-------------|-------|---------|----------|
| Like (Pink) on White | 4.2:1 | ‚úÖ Pass | Large text/icons only |
| Highlight (Orange) on White | 3.1:1 | ‚úÖ Pass | Icons/badges only |
| Positive (Green) on Surface | 5.2:1 | ‚úÖ Pass | Financial data |

### Color Blindness Support

- **Deuteranopia/Protanopia** (Red-Green): Avoid relying solely on red/green for critical information. Use icons and text labels.
- **Tritanopia** (Blue-Yellow): Primary blue and warning amber are distinguishable.
- **Best Practice**: Always pair colors with icons or text labels for status indicators.
